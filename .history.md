## 2026-06-08 CF Worker 海外IP触发上图风控问题分析

### 问题描述
项目部署到 Cloudflare Workers 后，Worker 访问上海图书馆服务器 (vufind.library.sh.cn) 触发了风控拦截。

### 假设
CF Worker 使用海外IP出口，上图WAF检测到海外IP后触发风控。

### 验证结果（假设已确认）
| 测试方式 | 出口IP | 结果 |
|---------|--------|------|
| 本地直连 | 中国IP | 正常返回2.3MB页面 |
| 通过代理 | 5.83.129.90（欧洲IP） | 302重定向到 /verification，要求通过腾讯防水墙验证码 |

**风控机制**：上图WAF检测到海外IP → 302重定向到 `/verification` → 加载腾讯防水墙(tac.js)验证码 → 提示"权限验证"/"您的验证过于频繁"

**CF Workers 出口IP**：全部是海外IP段（104.16.0.0/13, 172.64.0.0/13 等），必然触发风控。

### 提出的免费方案
1. **方案A：本地中转 + Cloudflare Tunnel** - CF Worker调用本地中转API，由国内IP请求上图
2. **方案B：纯前端方案** - CF只部署静态页面，浏览器直连上图（需CORS支持）
3. **方案C：国内云函数中转** - 用阿里云FC/腾讯云SCF免费额度做中转
4. **方案D：定时预取 + KV缓存** - 本地定时抓取存入CF KV，Worker只读KV

### 方案选择
- 纯前端方案不可行：上图无CORS头 + X-Frame-Options: SAMEORIGIN
- 选择方案C：国内云函数中转

### 云函数免费额度调研（2026-06-08）
| 平台 | 长期免费 | 额度 |
|------|---------|------|
| 阿里云FC | 否（仅3个月试用） | 15万CU/月×3个月 |
| 腾讯云SCF | 否（仅3个月试用） | 100万次/月×3个月 |
| **华为云FunctionGraph** | **是（永久）** | **100万次/月 + 40万GBs/月** |

**最终选择：华为云 FunctionGraph**，永久免费，额度可支撑约80万次/月请求。

### 实施进度
1. ✅ 编写华为云 FunctionGraph 中转函数代码 (cloud-function/index.js)
2. ❌ 华为云HTTP函数需要APIG专享版触发器（收费），无法免费获得URL
3. ❌ 阿里云FC/腾讯云SCF仅3个月免费，不满足长期免费需求
4. ✅ **转向桌面端方案**：直接从桌面端请求上图（国内IP不走中间层），零成本

- 历史条目: `{ id, keyword, search_type, result_count, created_at }`
- 收藏条目: `{ id, record_id, title, author, cover_url, created_at }`
- ID 使用 `Date.now() * 1000 + counter` 确保唯一性

## 2026-06-09 上海图书馆借阅排行榜API逆向分析

### 目标
探索上海图书馆"借阅排行榜"页面，逆向分析其API接口，为桌面端和Node.js版实现排行榜功能提供完整API文档。

### 页面信息
- 排行榜页面URL: `https://www.library.sh.cn/info/billboard`
- 技术框架: Nuxt.js SSR (Vue.js + 服务端渲染)
- 页面数据通过 `window.__NUXT__` 嵌入HTML，但实际数据通过AJAX API获取

### 逆向分析过程
1. 搜索找到排行榜页面URL
2. 下载页面HTML，识别Nuxt.js框架
3. 下载并分析JS bundle文件 (`js_c898528.js`, `js_2af7d17.js`)
4. 从JS源码中提取API调用逻辑和参数构造规则
5. 发现AAT Token认证机制
6. 逐步调试API参数格式（type拼接、date格式等）
7. 全面测试所有参数组合

### 关键发现

#### AAT Token认证
- 端点: `POST /st/token/aatTokenAcquire`
- 无需认证，POST空JSON `{}`即可获取
- 返回JWT token，有效期约28天
- Token必须作为请求体参数 `aat` 传递（非Cookie/Header）

#### type参数拼接规则（关键！）
JS源码中发现: `c.type = "".concat(c.type, "_").concat(c.dateType)`
- `type` = `{readerType}_{dateType}`
- 有效值: `adult_month`, `adult_year`, `children_month`, `children_year`

#### date参数格式
- 月榜: `"YYYYMM"` (如 `"202605"`)
- 年榜: `"YYYY"` (如 `"2025"`)
- 1月份取上年12月数据

#### 两个不同端点
- `queryBookBillboardGather`: 总榜（clc为空时使用）
- `queryBookBillboard`: 分类榜（clc有值时使用）

## 2026-06-09 排行榜点击详情页修复 + 返回按钮逻辑修复

### 问题1：排行榜书籍点击后无法正确跳转详情页

#### 根因分析
1. 排行榜API返回的`bookid`是bibid（如4783767），而VuFind详情页需要UUID格式的record ID（如55b6d3ff-6b5f-438e-8559-b1ebcb840613）
2. 之前用HTML解析方式从搜索结果页提取record ID，但VuFind搜索结果HTML中的链接使用HTML实体编码（`&#x2F;Record&#x2F;UUID`），正则无法匹配
3. Node版`index.ts`中`BASE_URL`未定义，导致lookup API请求URL变成`undefined/Search/Results?...`
4. 前端`showRankingDetail`中用`.ranking-title`选择器查找书名，但该选择器指向的是排行榜区域标题（"借阅排行榜"），不是书籍标题

#### 修复方案
**核心改动：改用VuFind JSON API替代HTML解析**

VuFind提供`/api/v1/search` JSON API端点，直接返回结构化数据（含record ID），无需解析HTML：
```
GET https://vufind.library.sh.cn/api/v1/search?lookfor=978-7-5192-0314-6&type=ISN&limit=1
→ {"status":"OK","resultCount":1,"records":[{"id":"55b6d3ff-6b5f-438e-8559-b1ebcb840613","title":"红墙图志 毛泽东"}]}
```

#### 修改文件
| 文件 | 修改内容 |
|------|---------|

### 问题3：少儿排行榜不显示

#### 根因
排行榜API的type参数中，少儿类型应为`child_month`/`child_year`，而非`children_month`/`children_year`。使用错误的`children_`前缀导致API返回code 500"该语句没有返回结果集"。

#### 修复
前端`loadRanking()`中拼接type时，将`children`映射为`child`：
```javascript
var readerKey = rankingState.readerType === 'children' ? 'child' : rankingState.readerType;
var type = readerKey + '_' + rankingState.dateType;
```

#### 修改文件
| 文件 | 修改内容 |
|------|---------|
| desktop/frontend/index.html | `loadRanking()`中children→child映射 |
| public/index.html | `getRankingType()`中children→child映射 |

### 问题4：程序图标不显示

#### 根因
之前的`rsrc_syso`文件没有`.syso`扩展名，Go编译器无法识别。Go的`go build`会自动将同目录下的`.syso`文件链接到exe中作为资源。

#### 修复
使用`rsrc`工具重新生成正确的`rsrc.syso`文件：
```
C:\Users\Administrator\go\bin\rsrc.exe -ico icon.ico -o rsrc.syso
```
